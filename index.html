Internal Strategy Discussion

과금 경험 전략
"돈을 쓰는 게 아니라, 결심을 하는 것"

단순한 코인 충전이 아닌, 결혼을 향한 진지한 선택의 순간으로 과금 경험을 재정의하기 위한 전략 아웃라인

결하다 Product Team · Design × PM Sync · 2026.03
01

왜 '클로버'가 위험한가

포인트/코인 구조가 결하다의 브랜드에 미치는 영향

결하다의 핵심 포지셔닝은 "가입비 없는 결정사"이고, 핵심 약속은 "운명이 아니라, 실패 확률을 낮춘 소개"입니다. 이 브랜드 위에 '클로버 충전'이라는 포인트형 과금이 올라가면, 사용자는 아무리 좋은 매칭 알고리즘이 있어도 이 서비스를 데이팅앱으로 인식합니다.

✕ 포인트/코인 구조가 주는 인상

"클로버 30개 충전"

게이미피케이션 → 가볍다 → 많이 써도 된다 → 여러 명 동시에 → 데이팅앱. 무의식적으로 '소비'의 프레임으로 전환됨. 결혼 상대를 만나는 행위가 '코인 소모'로 축소.

✓ 우리가 원하는 인상

"이 사람과 만남을 시작합니다"

결심 → 신중하다 → 한 명에 집중 → 진지한 관계 → 결혼중개. 과금의 순간이 곧 '이 사람과 진지하게 만나보겠다'는 의사 표현이 됨.

핵심 문제
결하다는 정찰제 ₩100,000입니다. 1회 매칭 수락 = 10만원. 이 구조에서 '포인트 충전'은 구조적으로 불필요합니다. 10만원짜리 결정을 '클로버 몇 개'로 환산하는 순간, 가격의 심리적 무게가 희석되고, 동시에 "얼마치를 충전해야 하지?"라는 불필요한 인지 부하가 생깁니다.
02

과금 전략: 3가지 방향

정찰제(₩100,000/1매칭)에 맞는 과금 구조 비교

💎
전략 A — 결심 기반 직접 결제
RECOMMENDED · 포인트 없이, 매칭 수락 시 직접 결제

포인트/코인 레이어를 완전히 제거합니다. 사용자가 매칭 리포트를 보고 "이 사람을 만나겠습니다"를 누르면, 그 순간 ₩100,000 결제가 일어납니다. 과금 = 결심의 순간.

장점
  • 결정사 포지셔닝과 완벽 정합
  • 가격 투명성 (정찰제 강조)
  • 인지 부하 최소 — 코인 환산 불필요
  • 결제 순간의 심리적 무게 유지
  • 데이팅앱 인상 완전 차단
고려사항
  • 매번 결제 프로세스 필요 (간편결제 연동 필수)
  • 번들 할인 설계가 별도 필요
  • 선결제/충전 대비 현금 유입 시점 늦음
📜
전략 B — 이용권 묶음 (바우처형)
"결이음권" 1장/3장/5장 선택 구매

포인트가 아닌 '이용권(바우처)' 개념으로 전환합니다. 클로버 같은 가상 화폐가 아니라, "3번의 만남 기회를 미리 준비한다"는 프레임. 묶음 구매 시 소폭 할인을 제공하되, 1장 단위 구매도 가능.

장점
  • 선결제로 현금 유입 조기화
  • 묶음 할인으로 재구매 유도
  • '포인트 충전' 느낌 없이 결제 가능
  • 이용권이라는 물리적 언어 → 신뢰
고려사항
  • 이름에 따라 여전히 '코인'처럼 느껴질 수 있음
  • 미사용 이용권 환불 정책 필요
  • 몇 장 사야 하지? → 약간의 인지 부하
🔄
전략 C — 하이브리드 (직접결제 + 이용권)
1회 직접결제 기본 + 묶음 이용권 옵션

기본은 전략 A(매칭 수락 시 직접 결제)로 운영하되, "진지하게 여러 분을 만나보고 싶은 분"을 위해 이용권 묶음 패키지를 별도 제공. 스토어 페이지에서 두 옵션을 모두 보여줌.

장점
  • 첫 사용자: 부담 없는 1회 결제 진입
  • 활성 사용자: 묶음 할인 혜택
  • 두 세그먼트 동시 커버
고려사항
  • UI 복잡도 증가
  • 초기 런칭에는 오버 엔지니어링일 수 있음
PM 의견
런칭 시점에는 전략 A(직접 결제)로 시작하고, 사용자 리텐션 데이터를 보고 전략 C로 확장하는 것을 권장합니다. 포인트 시스템은 결하다의 DNA와 충돌합니다.
03

네이밍 후보

과금 단위 / 결제 행위 / 스토어 이름에 대한 후보안

포인트를 제거하더라도, 결제 행위와 스토어에 부르는 이름은 필요합니다. 아래는 과금 단위(바우처형을 쓸 경우)와 결제 행위 자체의 네이밍 후보입니다.

A. 과금 단위 (바우처형을 쓸 경우)

결이음권
Gyeol-Ieum-Gwon — "Bond-Connecting Voucher"
결하다의 '결'과 이어준다는 '이음'을 결합. 이미 확립된 브랜드 언어이며, '권(券)'이라는 단어가 주는 공식적 느낌이 결혼중개업 맥락과 잘 맞습니다. "만남을 이어주는 공식 이용권"이라는 뉘앙스.
브랜드 정합성 ◎ 공식적 느낌 기존 사용 다소 길다
만남권
Mannam-Gwon — "Meeting Voucher"
가장 직관적이고 쉬운 표현. "만남을 시작하는 권리"라는 뜻이 즉시 전달됩니다. 다만 결하다만의 브랜드 고유성은 약합니다.
직관적 쉽다 브랜드 차별화 약함 범용적
결심권
Gyeol-Sim-Gwon — "Resolve/Decision Voucher"
'결심(決心)'에는 결하다의 '결'이 자연스럽게 포함됩니다. "이 사람을 만나겠다는 결심"이라는 의미 부여가 가능하며, 과금 행위에 감정적 무게를 실어줍니다.
의미 깊음 '결' 포함 결제 무게감 약간 무겁게 느껴질 수 있음
결매칭
Gyeol-Matching — "Decisive Matching"
가장 현대적이고 가벼운 느낌. '결(結)+매칭'으로 브랜드를 포함하면서도 쉽게 이해됩니다. "결매칭 1회 사용" 같은 표현이 자연스럽습니다. 다만 '권' 없이 단위로 쓸 수 있는지 검토 필요.
현대적 쉬운 발음 단위로 쓸 때 어색할 수 있음

B. 결제 행위의 네이밍 (버튼/UI 문구)

전략 A(직접 결제)를 사용할 경우, 버튼에 들어갈 문구가 곧 과금 경험의 핵심입니다.

피해야 할 표현

❌ 데이팅앱 느낌

"구매하기" · "코인 충전" · "클로버 사용" · "결제하기" · "포인트 차감" — 거래/소비의 언어

권장 표현

✓ 결정사 느낌

"만남 시작하기" · "이 분과 만나기" · "결이음 시작" · "만남을 열다" — 관계 시작의 언어

UX 카피 제안
CTA 버튼: "이 분과 만남 시작하기"
확인 팝업: "○○님과의 만남을 시작합니다. (₩100,000)"
결제 완료: "만남이 시작되었습니다 💐"
핵심 원칙: '결제'라는 단어를 사용자에게 직접 노출하지 않습니다.

C. 인앱 스토어 이름

결이음 센터
Gyeol-Ieum Center
스토어가 아니라 '센터'. 결하다의 Trust Center와 언어적으로 연결되며, "결이음을 시작하는 공간"이라는 느낌을 줍니다. 이용권 구매, 결제 내역, 환불 등 과금 관련 기능을 모두 포함.
만남 준비실
Meeting Preparation Room
좀 더 따뜻하고 오프라인 결정사 느낌. "만남을 준비하는 공간"이라는 메타포. 이용권 구매 외에 프로필 보완, 매칭 설정 등도 함께 배치하면 자연스러움.

05

가격 구조안 (바우처형 병행 시)

전략 C를 채택할 경우의 가격 테이블

구분 구성 가격 1회당 단가 할인율
단건 결제 만남 1회 ₩100,000 ₩100,000 —
결이음권 3 만남 3회 ₩270,000 ₩90,000 10%
결이음권 5 만남 5회 ₩400,000 ₩80,000 20%
가격 전략 포인트
단건 결제(₩100,000)가 기준가이자 앵커 프라이스. 묶음은 "여러 분을 진지하게 만나보고 싶은 분"에게만 제안합니다. 묶음 구매를 앱 메인에서 강조하지 않고, 결이음 센터(마이페이지) 안에서만 노출 → 코인 충전 느낌 방지.
06

클로버(🍀) 재설계 — "정성의 화폐"

과금과 분리된 비과금 리워드로, 상대방을 더 깊이 알아가는 데 사용

클로버를 과금 단위에서는 완전히 제거하되, "상대방을 더 깊이 이해하는 도구"로 전용합니다. 진지하게 활동하는 사용자에게 자연스럽게 쌓이는 보상이며, 쓸수록 더 나은 판단을 할 수 있게 도와줍니다.

🍀 클로버 획득 방법

진지하게 활동할수록 클로버가 자연스럽게 쌓입니다

기본 활동 보상
  • 프로필 완성도 100% 달성 → 🍀 ×2
  • 24문항 심리 분석 완료 → 🍀 ×1
  • 첫 매칭 리포트 확인 → 🍀 ×1
  • 인증 서류 제출 완료 → 🍀 ×1
참여 보상
  • 만남 후 양쪽 피드백 작성 → 🍀 ×2
  • 결하다 추천인 등록 → 🍀 ×3
  • 오프라인 결맞음 행사 참가 → 🍀 ×2
  • 월간 프로필 업데이트 → 🍀 ×1

🍀 클로버 사용처 — "더 깊이 알아보기"

매칭 수락 전후, 상대를 더 잘 이해하기 위한 도구들

💬

상대방 전체 답변 열람

🍀 ×3

24문항 심리 분석에서 상대가 직접 작성한 주관식 답변 전체를 확인합니다. 요약 리포트 너머, 그 사람의 실제 말투와 사고방식을 엿볼 수 있습니다.

📎

확장 프로필 해금

🍀 ×2

기본 프로필에 포함되지 않는 추가 정보를 확인합니다. 라이프스타일 상세, 가치관 에세이, 연애 히스토리 요약 등 심층 정보에 접근할 수 있습니다.

🎬

영상 프로필 열람

🍀 ×4

상대방이 등록한 자기소개 영상을 확인합니다. 사진만으로는 알 수 없는 분위기, 말투, 표정을 미리 느낄 수 있어 만남 결심에 가장 강력한 판단 근거가 됩니다.

🤖

AI 궁합 코치

🍀 ×3

AI가 나와 상대의 심리 분석 결과를 비교 분석해, 잘 맞을 포인트와 주의할 포인트를 구체적으로 조언합니다. "이 부분에서 갈등이 생길 수 있어요" 같은 맞춤 인사이트.

🔍

웨이브 심화 분석

🍀 ×2

기본 웨이브 리포트 너머, Big Five 세부 항목별 비교 차트와 매칭 타입 상세 해석을 확인합니다. 28가지 매칭 타입 중 내가 속한 타입의 관계 패턴 해설 포함.

💌

첫인사 도우미

🍀 ×2

만남 성사 후, AI가 상대 성향을 반영한 첫 연락 문구와 대화 주제를 제안합니다. "이 분은 이런 주제로 시작하면 자연스러워요" 같은 맞춤 가이드.

📊

나의 매력 리포트

🍀 ×3

내 프로필에서 상대방들이 가장 많이 본 항목, 매칭 수락률 추이, 나와 잘 맞는 타입 통계를 AI가 분석해 보여줍니다. 프로필 개선에도 활용 가능.

📝

만남 준비 가이드

🍀 ×2

만남 확정 후, 상대의 성향과 관심사를 기반으로 AI가 장소 추천, 대화 팁, 피해야 할 주제를 정리한 1:1 맞춤 만남 가이드를 제공합니다.

설계 원칙
클로버 사용처는 모두 "더 나은 판단"과 "더 나은 만남"을 돕는 것에 집중합니다. 프로필 상단 노출이나 더 많은 매칭 기회 같은 "경쟁" 요소는 의도적으로 배제합니다 — 결하다는 결을 맞추는 서비스지, 경쟁하는 서비스가 아닙니다.
클로버는 "돈"이 아니라 "정성"의 화폐입니다.
충전해서 사는 것이 아니라, 진지하게 활동해서 얻는 것.
그리고 쓰는 곳은 오직 상대를 더 깊이 이해하는 데에만.
수익 모델 연결 포인트
클로버 사용처 중 AI 기능(궁합 코치, 첫인사 도우미, 만남 준비 가이드, 매력 리포트)은 실제 API 비용이 발생합니다. 향후 클로버 부족 시 소액 구매 옵션(₩3,000~5,000)을 열면, 자연스럽게 부가 수익 채널이 됩니다. 핵심 과금(₩100,000 매칭)과 분리된 마이크로 과금 레이어로 기능합니다.
07

디자인 싱크 포인트

이번 미팅에서 결정할 것

Decision 1
과금 구조: 직접 결제(A) vs 바우처(B) vs 하이브리드(C)?
→ 런칭 시점에는 A를 권장, 이후 C로 확장
Decision 2
과금 단위 이름: 결이음권 vs 만남권 vs 결심권 vs 결매칭?
→ 직접 결제 시에도 영수증/내역에 표시될 명칭 필요
Decision 3
인앱 스토어 이름: 결이음 센터 vs 만남 준비실?
→ Trust Center와의 언어적 통일성 고려
Decision 4
클로버 사용처 우선순위: 8가지 중 런칭 시점에 어디까지 포함?
→ AI 기능(궁합 코치, 첫인사 도우미)은 API 비용과 개발 공수 고려
Decision 5
클로버 소액 구매 옵션: 런칭 시점부터 열 것인가, 추후 열 것인가?
→ "정성의 화폐" 포지셔닝과 충돌하지 않는 선에서 결정
Decision 6
CTA 문구: "이 분과 만남 시작하기" 등 구체적 UX 카피 확정
→ 디자인 팀과 워딩 테스트 필요
"돈을 쓰는 경험이 아니라, 관계의 다음 단계로 가는 결심의 순간으로."
모든 과금 UX의 기준은 이 한 문장입니다.
결하다 — 결이 맞아야, 결혼도 맞다.
